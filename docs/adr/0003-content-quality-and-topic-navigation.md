# 0003. 関連記事とコンテンツ品質検査を自動化し、記事の削除判断は監査に分離する

> **Status**: Accepted（関連記事の選定方式は [[0004-related-post-ranking]] で一部置換）

## Context（背景）
 - 全1752記事のうち本文500文字未満が584件、本文内の内部リンクがない記事が1607件あった
 - タグは表記揺れと単発利用が多く、従来のタグ一覧は約1.2MBあり全タグと全記事を一度に描画していた
 - 全記事一覧も約500KBあり、カテゴリごとに重複したレイアウトが保守されていた
 - 記事の自動削除や非公開化は既存URLと編集意図を損なうため、機械的には判断できない
 - ローカルのJekyll構成とGitHub Pagesの依存関係が異なり、push前にビルドと内部リンクを一貫して検査する仕組みがなかった
 - 上位方針は [[0001-seo-technical-scope]]、見出し規約は [[0002-heading-structure-normalization]] に従う

## Decision（決定）
 - 記事ページに共有タグを使った関連記事を最大5件表示する
 - 1つのタグから選ぶ関連記事は1件とし、同じ記事の重複表示を避ける
 - 公開タグ一覧と記事からのタグリンクは3記事以上で使われるタグに限定する
 - 大文字小文字だけが異なるタグと同じアンカーへ衝突するタグを正規化する
 - 全記事一覧とカテゴリ一覧はタイトルと日付を中心にした共通テンプレートへ統合する
 - `update_dates` は重複なしの新しい順、`sort_key` は公開日と更新日の最大値に正規化する
 - 記事の削除や非公開化は自動化せず、`docs/active/content-consolidation.md` に統合候補を生成する
 - GitHub Pagesと同じ `github-pages` gemを使い、`Gemfile.lock`を管理対象にする
 - 記事構造、メタデータ、タグ衝突、ローカル内部リンク、JekyllビルドをGitHub Actionsで検査する
 - MathJaxは数式を含むページだけで読み込む
 - `docs`、`scripts`、`AGENTS.md`などの運用ファイルはJekyllの公開対象から除外する

## 検討した代替案
 - 短文記事を一括削除する案はURL消失と判断ミスの影響が大きいため不採用
 - すべてのタグを公開し続ける案は単発タグが多く一覧が肥大化するため不採用
 - 本文へ関連記事リンクを一括追記する案は1752記事への恒久的な本文変更になるため不採用
 - 最新Jekyllへ直接更新する案はGitHub Pagesの管理ビルドとの差異が生じるため不採用

## Consequences（影響）
 - 個別記事から関連トピックへの導線が自動生成される
 - タグ一覧と全記事一覧の転送量とDOM要素数が減る
 - メタデータの新しい不整合と内部リンク切れをpush前に検出できる
 - 関連記事の選択はタグ品質に依存し、編集者が選ぶ導線ほど精密ではない
 - 本文500文字未満584件、概要がタイトルと同一147件、公開中のblog-drafts 15件は編集判断が必要な課題として残る
 - GitHub Pages固有の古いJekyll依存関係は維持されるため、将来独自Actionsデプロイへ移行する場合は別ADRで判断する
